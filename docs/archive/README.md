## 权威边界

- 本文件只说明归档区状态，不定义当前开发规则、架构事实或 API 契约。
- 当前开发事实应回到 `AGENTS.md`、`docs/README.md` 和对应稳定知识文档确认。

## 如何验证

- 检查归档文档是否在本文件登记，并带有清晰归档标记。
- 运行 `python3 scripts/validate_docs.py`，确认归档边界文档与本地链接仍然有效。

## 归档原则

1. **保留历史** - 不删除文档，移入归档目录
2. **标注状态** - 在归档文档顶部添加"已归档"标记
3. **合并价值** - 如有独特价值，先合并到权威文档再归档
4. **定期清理** - 每季度审查归档文件，删除无价值内容

---

## 归档流程

1. 确认文档已过时或被替代
2. 检查是否有独特价值需要保留
3. 如有价值，合并到当前权威文档
4. 移动文件到 `docs/archive/`
5. 在本 README 中添加记录
6. 在文档顶部添加归档标记
