# 数据权限与字段权限 PRD

## 背景

当前 DV-Admin 已具备接口级、菜单级和按钮级 RBAC：

- Django：`backend/drf_admin/utils/permissions.py` 的 `RBACPermission` 基于 `required_permissions` 做接口级校验。
- FastAPI：`fastapi/app/api/deps.py` 的 `PermissionChecker` 基于 `require_permissions` 做接口级校验。
- 前端：角色管理只维护菜单/按钮权限，`frontend/src/api/system/role-api.ts` 中 `dataScope` 仍是注释字段。

对标成熟后台系统后，长期可持续性缺口主要在数据范围与字段级访问控制：当前用户只要拥有接口权限，就能读取该接口返回的全部授权数据与字段。

## 目标

1. 为角色增加明确的数据范围配置，并由后端查询层强制执行。
2. 为高敏感字段提供读取隐藏/脱敏与写入拒绝能力。
3. Django 与 FastAPI 保持同一外部契约，前端不感知后端实现差异。
4. 保持接口级 RBAC 作为第一层校验，数据权限和字段权限作为第二层收敛。

## 非目标

- 不在本 PRD 阶段修改数据库模型或业务代码。
- 不用前端隐藏列替代后端强制权限。
- 不为所有模型一次性接入字段权限，先从用户、部门、角色、日志等高价值对象试点。
- 不引入临时 fallback 或绕过权限失败的静默降级。

## 数据权限模型

建议角色数据范围枚举：

| 值 | 名称 | 语义 |
|---:|------|------|
| 1 | 全部数据 | 不追加数据范围过滤 |
| 2 | 本人数据 | 仅允许访问 `owner/user_id/created_by` 指向当前用户的数据 |
| 3 | 本部门数据 | 允许访问当前用户所属部门数据 |
| 4 | 本部门及子部门数据 | 允许访问当前部门树下所有部门数据 |
| 5 | 自定义部门数据 | 允许访问角色绑定的部门集合 |

多角色合并规则建议采用“权限并集”：用户拥有多个角色时，取可访问部门集合并集；如任一角色拥有全部数据，则视为全部数据。

## 字段权限模型

建议字段策略：

- `readable=false`：响应中移除字段。
- `masked=true`：响应中保留字段但按字段类型脱敏。
- `writable=false`：写接口拒绝提交该字段，返回 403 或 400。

字段权限优先试点：

- 用户：`mobile`、`email`、`isSuperuser`、`roles`。
- 操作日志：`requestBody`、`responseBody`、`ip`。
- 通知：`targetUserIds`。

## 后端执行边界

后端必须作为唯一安全边界：

1. 接口权限先执行，失败直接拒绝。
2. 数据权限在 queryset/query builder 层追加过滤，不能只在 serializer/schema 后过滤。
3. 字段读取权限在 serializer/schema 输出前处理。
4. 字段写入权限在 request validation 后、业务写入前处理。
5. 超级管理员可按产品决策选择绕过数据范围，但仍应记录审计日志。

## 前端交互

角色表单建议增加：

- 数据范围下拉框。
- 自定义部门树，仅在数据范围为“自定义部门数据”时展示。
- 字段权限配置入口，按资源分组展示字段读/写/脱敏策略。

前端权限控制只用于交互提示和减少误操作，不作为安全边界。

## 契约与测试

需要新增或更新：

- `scripts/api_endpoint_contracts.py`：登记角色数据权限读写字段。
- `scripts/api_field_contracts.py`：登记角色表单新增字段。
- Django/FastAPI 模型契约：角色数据范围字段、角色-部门关系、字段权限表。
- Django/FastAPI 运行时测试：
  - 无权限不能越权读取其他部门数据。
  - 多角色数据范围按并集生效。
  - 字段不可读时响应不含字段或脱敏。
  - 字段不可写时写接口拒绝。
- 前端单元测试：
  - 角色表单按数据范围切换自定义部门树。
  - 字段权限配置提交结构稳定。

## 分阶段实施

### 阶段 1：数据范围最小闭环

- 给角色增加 `dataScope`。
- 给角色增加自定义部门关系。
- 用户列表和日志列表先接入数据范围过滤。
- 补双后端运行时越权测试。

### 阶段 2：字段读取权限

- 建立字段权限目录。
- 用户与日志列表先接入字段隐藏/脱敏。
- 补字段响应契约和前端展示测试。

### 阶段 3：字段写入权限

- 写接口增加字段写入拒绝。
- 用户、角色、通知写接口优先接入。
- 补越权写入测试。

### 阶段 4：治理自动化

- 将数据权限/字段权限接入契约报告。
- 在文档中标注已覆盖资源和暂未覆盖资源。

## 风险

- 数据范围语义需要产品确认，尤其是“本人数据”如何映射不同业务表。
- 字段脱敏策略需要统一，否则会造成前后端展示不一致。
- 双后端模型和迁移必须同步，否则会重新引入能力漂移。

## 验收标准

- 数据范围由后端强制，不能通过手写请求绕过。
- 字段权限由后端强制，不能通过前端 DevTools 恢复。
- Django 与 FastAPI 对同一用户、同一角色配置返回一致结果。
- API 契约、字段契约、模型契约和运行时测试均通过。
